静态内部类

如果不需要内部列对象与外部类对象之间有联系，可以使用静态内部类。创建静态内部类意味着：

  • 创建静态内部类，不需要持有外部类的引用
  • 不能从静态内部类中访问非静态的外部类对象

为什么要内部类

一般来说，内部类继承自某个类或者实现某个接口，内部类的代码操作创建它的外围类对象，所以可以认为内部类提供了某种进入其外部类的接口。

如果只是需要持有一个接口的引用，则直接通过外部类实现即可，那么内部类实现一个接口和外部类实现一个接口有什么区别呢？答案是后者不是总能享受到接口带来的方便，有时需要接口的实现，所以使用内部类最重要的原因是：

每个内部类都能独立的继承一个实现，所以无论外部类是否继承了某个实现，对于内部类而言是没有影响的。如果没有内部类提供的，可以继承多个具体的或者抽象类的能力一些设计与编程问题很难解决，内部类使得多重继承的解决方案变得完整。

如果不用解决多重继承的问题，可选择的实现方式很多，自然不需要使用到内部类。但是使用内部类，能够获得一些其他的特性：

  1. 内部类可以有多个实例，每个实例都有自己的状态信息，并且与其外部类对象的信息相互独立
  2. 在单个外部类，可以让多个内部类以布偶同德方式实现同一个接口，或者继承同一个类
  3. 创建内部类对象的时候并不依赖与外部类对象的创建
